| 术中低血压是非心脏手术全身麻醉过程中的高发并发症，持续的低血压状态会显著升高术后急性肾损伤、心肌损伤等器官不良事件的发生风险，提前预警并采取干预措施对改善患者围术期预后具有重要临床意义。现有低血压预测模型多依赖有创动脉压高频波形数据，临床部署门槛较高；而基于常规监护参数的模型普遍存在长程时序特征捕捉不足、极端样本不平衡下少数类召回率偏低的问题。针对上述局限，本研究基于 VitalDB 围术期生理数据集，以 10 分钟生理观测窗口预测未来 10 分钟低血压事件为目标，筛选 16 项核心动态生理参数并结合 3 项血压趋势衍生特征构建模型输入，搭建了融合空洞卷积残差网络与 Transformer 编码器的混合深度学习架构。模型通过带通道注意力的卷积模块提取局部多尺度生理变化特征，依托 Transformer 全局自注意力机制建模长时序依赖关系，同时采用 Focal                       |
| 损失函数与加权随机采样双重策略缓解正负样本极端不平衡的影响。实验结果表明，在正样本占比仅 7.9% 的测试集上，模型 ROC 曲线下面积达 0.9366，精确率 - 召回率曲线下面积达 0.6477，最优分类阈值下 F1 值为 0.6275，低血压事件召回率达 0.60。该模型仅依靠常规监护设备的稀疏数值参数即可实现较为精准的低血压提前预警，无需额外有创监测设备，具备较好的临床应用潜力。                                                                                                                                                                                                                                                                                                                                                                                                     |

| **1.研究背景**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 术中低血压是全身的副麻醉常见作用，在非心脏手术中术中低血压（IOH）多见，且与严重肾脏、神经系统、心脏不良事件甚至死亡相关 \[1\]。术中心输出量下降、心肌收缩力降低、失血、麻醉药物引发血管扩张等多种因素均可诱发 IOH \[2\]。平均动脉压（MAP）可用于评估组织灌注水平，研究表明MAP＜80 mmHg 且持续≥10 min 即可诱发组织灌注不足、造成器官缺血损伤。MAP＜70 mmHg 短期暴露同样存在损伤风险，MAP＜65mmHg时器官损伤风险会随低血压持续时间明显升高，临床常将 MAP＜65mmHg作为术后器官损伤的关键临界值\[1\]。机体血压稳态主要依靠压力感受器反射调节心率与血管收缩，并通过交感-肾上腺系统分泌升压物质维持血压稳定\[2\]。低血压预测指数等监测预测技术能够提前预判低血压发作，并配套相关参数辅助临床医师明确 IOH                                                               |
| 病因、指导干预。现有的IOH预警模型大多只利用时域信息而忽略频域，长时序模型少且效果不佳，对噪声敏感容易受到干扰，ZengX等人构建的SAFDNet\[4\]模型采用了多模态输入，结合真实的临床场景多样化数据，通过引入傅里叶变换模块提取频域特征和自适应阈值过滤噪声，将只通过波形预测血压转变为了分析频谱结构。并且建立了双路径注意力机制，利用CNN捕抓短期变化，LSTM捕抓长期趋势。Mohammadi I, Rajai F S也尝试同时将ABP、ECG、PPG、ETCO₂四通道波形 + 术前临床特征作为输入利用CNN+RNN（GRU，深度学习模型）提前五分钟预测IOH变化，该模型比目前商用的HPI（单一动脉波形模型）更鲁棒、更适合真实手术室。\[5\]根据最新的研究和观察，术中低血压的发生在非心脏手术患者中是相当常见的。例如，一项回顾性多中心观察研究显示，术中平均动脉压低于65                                        |
| mmHg的持续时间为28.2分钟，88%的病例至少出现一次低血压事件。此外，另一项研究指出，术中低血压的发生率在各个医疗中心之间存在显著差异，从83.2%到91.6%不等。这些数据表明，术中低血压仍然是手术中一个重要的临床问题，需要引起足够的重视和有效的管理。利用深度学习和机器学习，对IOH事件进行提前预测是近年来医学和工程领域的热门课题之一，这能使得医生有更多的时间处理IOH事情的发生，具有重要意义。                                                                                                                                                                                                                                                                                                                                                                    |

| 3.1标签的构造                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 对vitalDB数据集进行去除MBP异常数据点线性插值，对血流动力学参数的缺失数据进行线性插值保存为vitalDB_Pre_Process_03。采用基于面积的低血压事件标签定义方式。具体而言，标签构造基于预测窗口内的MAP相对于65 mmHg 参考阈值偏离程度进行量化。累积面积量$E$通过对预测窗口内每一时间步的$\left( 65 - \text{MAP}_{t} \right)$逐段累加并乘以窗口粒度Δt获得，其计算公式为：                                                                                                                                                                                                                                                                                                                                                                                                 |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| $E = \sum_{t = 1}^{T_{\text{pred}}}{(65 - {MAP}_{t})}$ （3-1）                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 在公式的计算逻辑中，当某一时间步的$\text{MAP}_{t}$低于65 mmHg时，该时段产生正向数值贡献，表示低灌注暴露在该时间段内继续累积；当$\text{MAP}_{t}$回升并高于65mmHg时，相应项则对净面积产生抵消作用。因此，累积面积量E同时反映低血压暴露的深度信息与持续时间信息。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 得到连续度量E后，进一步将其转化为二分类标签。以Mukkamala研究综述\[6\]建议的30分钟预测窗口对应阈值75$\text{mmHg} \cdot \text{min}$作为参照基准，判定阈值$\tau$的计算公式定义如下：                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| $\tau = 75 \times \frac{L_{pred} \cdot \Delta t}{30min}$ （3-2）                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 本文在构建标签的时，窗口粒度设定为0.5min，观察窗10min，预测窗 10min，滑动步长 1min。当 E \> τ 时，标签设为 1（阳性，预测未来将发生显著低血压）；否则设为 0（阴性）。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

| 从vitalDB数据集中，经过系统的特征筛选后保留16个核心动态特征用于模型输入。筛选的特征需满足生理相关性与血流动力学调节机制直接相关、在多数患者记录中具有足够的有效采样率和去冗余性排除与已有特征高度相关的参数。如无创血压参数因与有创血压高精度冗余和麻醉深度指标与低血压发生的因果链距离较远而被排除。丙泊酚、瑞芬太尼、罗库溴铵等药物输注速率因在时间上通常晚于低血压发生为干预的结果而非前兆而被排除，以避免数据泄漏。                                                                                                                                                                                                                                                                                                                                        |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 保留的16个动态特征包括5项核心血流动力学参数、2项氧合与通气参数、4项呼吸力学参数、2项气道压力参数，以及3项麻醉状态与体温参数。参数具体含义见附录表X。在此基础上，从观察窗口内的MBP轨迹中额外提取3个标量衍生特征，以显式编码血流动力学的趋势信息。包括趋势斜率通过线性回归拟合观察窗口内20个时间步的MBP序列得到的MBP、MBP低于65mmHg的时间占比，以及MBP波动幅度即观察窗口内MBP的最大值与最小值之差，反映血流动力学的稳定性。这3个标量被复制20次以匹配时间步维度，与16个动态特征拼接，构成最终的19维特征向量。因此，每个样本的形状为20×19。                                                                                                                                                                                                                        |

| **4.模型设计与构建**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 在仅给定20个时间步（30秒间隔）的情况下，从19个多源异构生理参数中提取有效的时空特征，以判别未来十分钟的低血压风险。这一任务对特征提取器提出了三个要求即，能够捕获各参数内部短时间尺度上的局部变化模式，能够建模不同参数之间的跨通道交互关系，能够整合整个观察窗口长度的全局时序依赖。因此本文的CNN-Transformer混合架构需要分别对上述三种需求设计专门的模块。\[9\]一维卷积残差网络负责局部时空特征的提取，通过逐层扩大的空洞卷积感受野捕获从单次心跳到分钟尺度的生理变化模式，并利用通道注意力机制（Squeeze-and-Excitation，SE）自适应地为不同生理参数分配重要性权重。\[8\]Transformer编码器负责全局时序依赖的建模，通过多头自注意力机制在任意两个时间步之间建立直接的交互通路，不受卷积感受野的局部性限制。最后，多头注意力池化层在时间维度上自适应地聚合信息。 |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 4.1输入预处理与特征增强                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 在进入核心编码器之前，输入特征经过三步预处理。首先，利用预先标定的各特征生理范围进行最小-最大归一化，将每个特征映射到近似的\[0,1\]区间。其次，通过计算各特征沿时间维度的一阶差分和二阶差分，显式地为模型提供变化速率和变化加速度信息。这两类差分特征与原始特征在通道维度拼接，使输入从19维扩展至57维。差分特征的引入减轻了模型从原始值中隐式学习变化趋势的负担，通过可学习的位置编码向量与每个时间步的特征相加，为Transformer编码器提供时间位置信息。                                                                                                                                                                                                                                                                                                          |

| 4.2 卷积残差编码器                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 编码器由两个残差块级联而成，每个残差块包含两个一维空洞卷积层、批归一化、ReLU激活和SE通道注意力模块。第一个残差块使用膨胀率为1的卷积（标准3×1卷积，感受野局限在相邻时间步），将57维输入映射至192维的隐藏表示。第二个残差块使用膨胀率为2的卷积（核内各元素间距为2，感受野扩大但参数量不变），将通道数扩展至384维。残差连接使得梯度可以直接从深层回传至浅层，缓解了深度网络的梯度消失问题，同时允许网络学习恒等映射------当某个特征通道对当前任务不重要时，SE模块可以将其压缩为零，残差连接则保留原始信息不受影响。                                                                                                                                                                                                                                               |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| SE注意力模块的运算流程为：对每个通道沿时间维度取全局均值，通过一个瓶颈结构的全连接层学习各通道的重要性权重，最终将该权重乘回原始特征图。这一机制允许模型在训练过程中自动发现哪些生理参数对低血压预测最为关键，并相应地为这些通道分配更高的权重，同时抑制不相关参数的噪声。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |

| **5.模型训练**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| vitalDB数据集中，正负样本分布存在较为明显的不平衡现象。\[11\]若直接采用普通二分类交叉熵损失函数进行训练，模型容易受到多数类样本的影响，从而降低对少数类样本的识别能力，表现为低血压事件召回率偏低。针对这一问题，采用Focal                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| Loss作为模型的损失函数，通过增加困难样本的权重，使模型更加关注少数类样本的学习。\[10\]在参数设置上，正样本权重取0.75，负样本权重取0.25，聚焦参数γ设置为4，同时引入0.05的标签平滑策略，以减轻模型过拟合和预测过于自信的问题，提高训练稳定性。在数据采样方面，为进一步缓解类别不平衡带来的影响，训练阶段采用加权随机采样策略。训练时保留全部正样本，并按照正负样本1∶2的比例随机抽取负样本，使每个批次中正样本所占比例得到提高，从而增强模型对低血压事件特征的学习能力。而在验证集和测试集阶段，不进行任何重采样操作，保持数据的真实分布，以保证模型评估结果能够客观反映实际应用场景下的预测性能。                                                                                                                                                                |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 模型训练采用AdamW优化算法，初始学习率设为3×10⁻⁴，权重衰减系数设置为1×10⁻³，以降低模型过拟合风险。训练初期设置3个Epoch的线性学习率预热（Warm-up），使学习率逐渐增大到设定值，避免参数更新过快导致训练不稳定。完成预热后，采用余弦退火重启（Cosine Annealing Warm Restarts）策略动态调整学习率，每8个Epoch完成一次学习率周期变化，从而兼顾模型的收敛速度和最终性能。                                                                                                                                                                                                                                                                                                                                                                                             |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 为了提高模型的泛化能力，在网络结构中加入了Dropout正则化。残差卷积模块和Transformer编码器的Dropout率设置为0.15，分类器部分提高至0.30，以减少神经元之间的共适应现象。同时从第30个Epoch开始引入随机权重平均策略，对训练后期多个模型参数进行平均，并在训练结束后利用平均权重进行模型推理，从而获得更加平滑、稳定的模型参数，提高模型在未知数据上的泛化性能。                                                                                                                                                                                                                                                                                                                                                                                                       |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 数据集划分采用患者级划分方式，避免数据泄露。首先按照患者编号进行随机划分，将80%的患者作为训练集，其余20%的患者作为验证集，同一患者产生的全部样本仅属于其中一个数据集，避免同一患者的数据同时出现在训练集和验证集中，从而有效防止数据泄漏，使模型验证结果能够更加真实地反映其对未知患者的泛化能力。                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 模型性能评估以ROC-AUC作为主要评价指标，并同时统计PR-AUC、F1值以及对应的最优分类阈值。为获得最佳分类效果，本文采用网格搜索方法，在0.05～0.95范围内以0.01为步长遍历所有候选阈值，计算各阈值对应的精确率、召回率和F1值，最终选择F1值最大的阈值作为模型最终的分类阈值，以实现精确率与召回率之间的较好平衡。                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **6.模型训练结果**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 在测试集的最终性能评估中，所提模型在总计16814例样本（含低血压正样本1333例，样本占比约7.9%具体数据见附录表Y）的极端类别不平衡条件下，取得了 0.9366的 ROC 曲线下面积（ROC-AUC）与0.6477的精确率-召回率曲线下面积（PR-AUC），反映出模型具备优异的整体判别能力，且 PR-AUC                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| 指标进一步验证了模型在少数类识别场景下的有效性。当分类阈值取最优值0.640时，模型的F1值达到最优的0.6275。对应混淆矩阵显示，真阴性样本为15060例，假阳性421例，假阴性531例，真阳性802例。分类别指标统计表明，阴性样本的精确率、召回率与F1-score均达0.97，维持了极高的识别准确度；阳性样本的精确率为0.66，召回率为0.60，F1-score为0.63。模型整体分类准确率为0.94，宏平均下精确率、召回率与F1-score分别为0.81、0.79与0.80，加权平均三项指标均为0.94。上述结果验证了所采用的CNN-Transformer混合架构FocalLoss、加权采样相结合的类不平衡处理策略的有效性，能够在极端样本分布下避免模型向多数类偏置，在保障正常样本高判别精度的同时，实现对低血压事件较为可靠的提前预测。                                                                                                |

| **7.讨论与总结**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 本研究针对术中低血压提前预警的临床需求，基于稀疏采样的多源生理参数构建了CNN-Transformer混合预测模型，实现了10分钟观测窗口下预测未来10分钟IOH事件的发生。从测试集性能来看，模型在正样本占比不足8%的极端不平衡条件下取得了0.9366的 ROC-AUC，整体判别能力优于同输入模态下的现有主流方法，验证了所提架构与优化策略的有效性，同时也反映出多尺度特征融合与全局时序建模在生理时序预测任务中的重要价值。                                                                                                                                                                                                                                                                                                                                                               |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 从模型设计的内在逻辑来看，混合架构的模块化设计精准匹配了术中低血压的生理特征规律。术中血压的波动既包含心跳、呼吸周期带来的短时高频波动，也包含药物代谢、失血累积带来的长时渐进趋势，单一网络结构难以同时覆盖两类特征。本研究中，空洞卷积残差网络负责捕捉分钟级以内的局部参数突变与跨参数耦合关系，SE通道注意力则自动锚定MBP、HR等核心血流动力学指标。而 Transformer 编码器则突破了卷积的局部感受野限制与RNN的长程衰减缺陷，能够建立观察窗口内任意时间点的直接关联，有效捕捉药物效应缓慢消退、代偿机制逐步失效等长时程前驱信号。此外，显式构造的一阶与二阶差分特征降低了模型学习趋势信息的成本，是稀疏采样条件下维持模型性能的关键设计之一。                                                                                                                    |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 针对类不平衡问题的双重优化策略是本研究的另一项核心设计。术中低血压事件天然属于低概率事件，标准交叉熵损失极易导致模型向多数类偏倚，出现高准确率、低召回率的结果，这在临床预警场景中是不可接受的。因此本文模型通过Focal Loss通过聚焦难分类样本与提升正样本权重，从梯度层面修正模型的学习偏向同时加权随机采样则从数据分布层面提升批次内正样本占比，避免模型在训练初期被大量负样本主导。从最终结果来看，阴性样本的精确率与召回率均保持在0.97的高水平，同时阳性样本召回率达到0.60，二者实现了较好的平衡，说明该策略既没有为了提升少数类识别而牺牲多数类的精度，也没有因数据不平衡而出现严重的漏诊，能够适配临床场景中"漏诊风险高于误诊" 的实际需求。                                                                                                                |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 与现有研究横向对比，本模型的性能表现具备明确的相对优势。与传统机器学习方法 XGBoost、纯数值输入的 Transformer 基线模型相比，本研究的ROC-AUC提升幅度达 4.5\~7.5个百分点，\[14\]证明深度学习混合架构在多参数时序特征提取上的显著优势。与仅依靠有创动脉压波形的单模态模型相比，本研究仅通过30秒间隔的数值参数便实现了更优的整体性能，\[12\]体现了多生理参数融合的信息增益。当然，与采用多通道高频波形输入的SOTA模型相比，本研究的整体性能仍存在一定差距，但这一差距主要来源于输入信息密度的差异，高频波形包含脉搏波形态、脉压变异度等丰富细节，而本模型的输入仅为降采样后的数值趋势。                                                                                                                                                                              |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 本文研究中也存在一定的局限性。首先，本研究的数据全部来源于VitalDB单中心公开数据集，样本以普通非心脏手术为主，模型在心脏手术、急诊手术、儿科患者等特殊场景中的泛化能力仍有待验证。其次，当前仅纳入了术中动态生理参数，未结合患者年龄、基础疾病、术前用药、手术类型等静态临床信息，而上述因素是低血压发生的重要风险因素，纳入后有望进一步提升模型的预测精度。并且深度学习模型的黑箱特性使得预警结果缺乏可解释性，医生无法直接获知引起IOH的因素。最后，当前标签采用累积面积定义，不同研究间的窗口长度、阈值设定存在差异，一定程度上影响了跨研究的结果可比性。                                                                                                                                                                                                     |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **8.未来研究**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
|                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 本研究的数据全部来源于VitalDB单中心公开数据集，将来需要在更多的公开数据集上进行验证以确定模型是否具鲁棒性和延伸性。并融入血流动力学相关的物理机制，从而解决深度学习模型的黑箱特性使得预警结果缺乏可解释性问题，使得医生不仅能得知预测结果还能获取引起IOH事件发生的因素，以此在极短时间内进行处理。同时在当前多模态模型的基础上融入患者年龄、基础疾病、术前用药、手术类型等与低血压发生存在相关性的静态临床信息，有望进一步提升模型的预测精度。                                                                                                                                                                                                                                                                                                                 |
